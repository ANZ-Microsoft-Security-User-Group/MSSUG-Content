# ANZ Microsoft Security User Group

The **ANZ Microsoft Security User Group** is a practitioner‑led community focused on sharing real‑world knowledge and experience across the Microsoft Security ecosystem.

This group brings together security professionals from across **Australia and New Zealand**—including defenders, architects, engineers, and decision‑makers—who are actively designing, deploying, and operating Microsoft security solutions in production environments.

## Purpose of this Repository

This GitHub repository is used to **share content with attendees** of the ANZ Microsoft Security User Group, including:

- Session slides and supporting materials  
- Demo scripts, sample configurations, and reference artefacts  
- Links to relevant Microsoft Learn and official documentation  
- Follow‑up resources discussed during meetups or workshops  

The intent is to provide **practical, reusable material** that attendees can take back to their own environments, rather than polished marketing assets.

## Community Principles

Content shared here is guided by a few simple principles:

- **Practitioner first** – focused on real deployments, lessons learned, and operational reality  
- **Vendor‑accurate** – aligned to current Microsoft product capabilities and official guidance  
- **Open and collaborative** – shared to support learning, discussion, and peer improvement  
- **Safe for customers** – no internal-only material, confidential data, or unsupported claims  

## Scope

Topics typically covered include (but are not limited to):

- Microsoft Defender (XDR, Endpoint, Identity, Office 365, Cloud Apps)  
- Microsoft Sentinel and security operations  
- Microsoft Purview and data protection  
- Entra ID, Conditional Access, and identity security  
- Secure access for unmanaged or hybrid environments  
- Emerging capabilities across the Microsoft Security stack  

## Disclaimer

All content in this repository reflects the **experience and perspectives of the presenters**, not official product commitments or roadmaps.  
Attendees should always validate configurations and recommendations against **current Microsoft documentation** before applying them in production environments.
